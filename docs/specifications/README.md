# 仕様書

このディレクトリには、プロジェクトの機能仕様や設計ドキュメントを保管します。

## ファイル構成

機能や設計領域ごとにファイルを分けて管理することを推奨します：

- `overview.md` - プロジェクト全体の概要
- `architecture.md` - アーキテクチャ設計
- `commands.md` - コマンド仕様
- `configuration.md` - 設定項目仕様
- `api.md` - API設計
- `ui.md` - UI/UX設計
